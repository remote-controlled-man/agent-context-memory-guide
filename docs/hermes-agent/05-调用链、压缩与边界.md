# `hermes-agent`：调用链、压缩与边界

## 1. 初始化主链

主调用链的骨架是：

`run_agent.py` 初始化 `MemoryStore`、`MemoryManager`、`ContextCompressor`，再在 build prompt、prefetch、tool dispatch、turn end 等节点把这些能力串起来。

### 源码锚点

- `run_agent.py:1219-1320`
- `run_agent.py:8237-8595`
- `run_agent.py:11329-11332`

## 2. 压缩主链

压缩路径是：

`flush_memories()` -> provider `on_pre_compress()` -> `ContextCompressor.compress()` -> 重建 system prompt -> 新 session。

这里真正被改变的不只是消息列表，还包括 prompt snapshot 与 session lineage。

### 源码锚点

- `run_agent.py:7040-7263`
- `agent/context_compressor.py:336-1094`

## 3. 历史检索主链

`session_search` 会排除当前 lineage，对命中的 transcript 片段做截断与总结，再作为可读回忆返回。这条链路与 provider recall 并列存在，而不是 provider recall 的一种实现。

### 源码锚点

- `tools/session_search_tool.py:245-560`

## 4. 关键边界

- `session_search` 不是 long-term provider memory。
- 内建 `MEMORY.md / USER.md` 不是 session transcript。
- provider memory 与 prompt builder 并列，不应并入“单一 memory 层”。

### 源码锚点

- `tools/session_search_tool.py:245-560`
- `tools/memory_tool.py:105-571`
- `agent/prompt_builder.py:144-173`
