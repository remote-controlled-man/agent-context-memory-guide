# `hermes-agent`：架构与范围

## 1. 三条主干

`hermes-agent` 与本任务最相关的三条主干是：

- `prompt_builder + run_agent`：system prompt 与 turn 上下文组装。
- `hermes_state + gateway/session`：session transcript 的 SQLite / JSONL 持久化。
- `memory_tool + memory_manager + plugins/memory/*`：内建 memory 与外部 provider memory。

### 源码锚点

- `run_agent.py:1219-1320`
- `agent/prompt_builder.py:144-173`
- `hermes_state.py:115-1199`
- `gateway/session.py:498-1045`
- `agent/memory_manager.py:57-356`

## 2. 范围边界

本组文档重点覆盖：

- `run_agent.py`
- `agent/*`

- `hermes_state.py`
- `tools/session_search_tool.py`
- `gateway/session.py`

不把外围 skills、网站资源或无关工具实现当作 memory 主线。

### 源码锚点

- `tools/session_search_tool.py:245-560`
- `plugins/memory/__init__.py:66-404`

## 3. 关键边界

- `session_search` 是历史 transcript 检索，不是 provider memory 本体。
- 内建 memory 和外部 provider memory 同时存在，但不是一回事。
- context compression 是单独的一条主线，不应简化成“摘要提示词”。
