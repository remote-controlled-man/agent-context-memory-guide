# `hermes-agent`：存储与状态

## 1. `state.db`

`hermes_state.py` 定义了 SQLite `state.db` 的 session / message / FTS5 结构，并承担标题、分支、压缩后新 session 等元数据持久化。

### 源码锚点

- `hermes_state.py:115-1199`

## 2. gateway transcript 双写

`gateway/session.py` 会把消息同时写进 `sessions/<session_id>.jsonl` 和 SQLite；`load_transcript()` 会比较 JSONL 与 DB 哪边消息更多，避免迁移或异常情况下的历史截断。

### 源码锚点

- `gateway/session.py:947-1045`
- `gateway/mirror.py:1-124`

## 3. 本地 memory 文件与 provider 配置

内建 memory 落在 `HERMES_HOME/memories/`；不同 provider 又各自维护配置文件或本地工作目录，例如 `supermemory.json`、`mem0.json`、`honcho.json`、`byterover/`。

### 源码锚点

- `tools/memory_tool.py:105-198`
- `plugins/memory/supermemory/__init__.py:1-250`
- `plugins/memory/mem0/__init__.py:1-120`
- `plugins/memory/honcho/__init__.py:1-160`
- `plugins/memory/byterover/__init__.py:1-120`

## 4. `session_search` 的状态来源

`session_search_tool.py` 直接读取 `state.db` 的 transcript 历史并走 FTS5 检索，再把命中的 session 摘要化返回。因此它的底座是历史数据库，而不是 provider memory backend。

### 源码锚点

- `tools/session_search_tool.py:245-560`
