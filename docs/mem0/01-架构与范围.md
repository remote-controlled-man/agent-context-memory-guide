# `mem0`：架构与范围

## 1. 主体结构

Python OSS 核心里最重要的模块有：

- `Memory / AsyncMemory`
- `storage.py` 的 SQLite 历史与消息窗口
- `utils.py` 的消息预处理
- `configs/prompts.py` 的抽取 prompt
- `utils/factory.py` 的 embedder / llm / vector store / reranker 工厂

- `mem0/memory/main.py:268-1687`
- `mem0/memory/storage.py:11-257`
- `mem0/memory/utils.py:61-170`
- `mem0/configs/prompts.py:947-1016`
- `mem0/utils/factory.py:30-30`

## 2. 范围边界

本组文档主分析对象只覆盖 Python OSS 核心 `mem0/`。`mem0-ts`、`server`、`vercel-ai-sdk`、`openmemory` 只在附录或横向对照里顺带点到，不按同等深度展开。

### 源码锚点

- `mem0/__init__.py:1-1`

## 3. 关键边界

- `context` 主要服务 memory 抽取，而不是完整对话 runtime。
- SQLite `history/messages` 不是 long-term memory 本体。
- 向量库 + entity store 才是长期记忆主体。
