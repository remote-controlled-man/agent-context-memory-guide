# `langmem`：架构与范围

## 1. 模块分层

`langmem` 的主干可以按四层理解：

- `short_term`：短期上下文摘要。
- `knowledge`：长期 memory 抽取、搜索、写回与工具封装。
- `reflection`：后台或远端执行 memory 处理。
- `utils / graphs`：namespace、消息格式化、auth 隔离、LangGraph 挂载。

- `src/langmem/knowledge/extraction.py:217-2109`
- `src/langmem/reflection.py:90-401`

## 2. 范围边界

本项目不是单个 agent runtime，没有仓库自定义的本地 SQLite / transcript 层。`langgraph.json` 只负责 graph、auth 和 store 挂载，真正的存储依赖注入式 `BaseStore`。

### 源码锚点

- `langgraph.json:3-11`
- `src/langmem/graphs/semantic.py:13-41`

## 3. 关键边界

- `running_summary` 是 graph state，不是长期 memory。
- `BaseStore namespace` 是运行时解析，不是固定目录路径。
- `graph_rag.py` 当前只是注释示意，不属于主运行链。
