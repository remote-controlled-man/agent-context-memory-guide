# `langmem`：调用链、压缩与边界

## 1. 三条主调用链

- `messages -> summarize_messages / SummarizationNode -> context.running_summary`
- `messages + existing memories -> MemoryManager -> MemoryStoreManager -> BaseStore`
- `messages -> ReflectionExecutor -> background runnable / remote run`

### 源码锚点

- `src/langmem/short_term/summarization.py:337-860`
- `src/langmem/knowledge/extraction.py:217-1666`
- `src/langmem/reflection.py:143-401`

## 2. 压缩边界

`langmem` 的“压缩”主要就是摘要节点本身，而不是完整 agent runtime 里的 transcript compaction。它处理的是“如何把旧消息压成继续可用的 summary”，不是“如何重写 thread 历史文件”。

### 源码锚点

- `src/langmem/short_term/summarization.py:102-334`

## 3. 关键边界

- `graph_rag.py` 当前不是主链。
- 许多行为依赖测试确认，例如 tool calls、重复 ID、same-key overwrite。
- 这是一套 memory/context 组件，不负责完整 agent 生命周期。
