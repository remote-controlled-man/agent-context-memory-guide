# `codex`：架构与范围

## 1. 三层主干

`codex` 的相关实现可以按三层理解：

- `core`：组装 turn 级可见 context，驱动采样、compact、memory startup。
- `state`：把 thread、memory jobs、stage1 outputs、spawn edges 等元数据落到 SQLite。
- `core/memories`：定义 phase1 / phase2 memory 管线，并把结果同步到文件系统 memory root。

## 3. 关键边界

- `ContextManager` 不是“历史文件”，而是 turn 级历史容器。
- `history.jsonl` 不是 SQLite state。
- `memories/` 目录不是唯一真相，SQLite `stage1_outputs` 与 `jobs` 也保存 memory 生命周期。
- `memory tool` 的读路径只看 `memory_summary.md`，不会直接扫整个 `memories/` 目录。
