# `codex`：调用链、压缩与边界

## 1. turn 级主调用链

主链路可以概括为：

`run_turn()` -> `record_context_updates_and_set_reference_context_item()` -> `clone_history().for_prompt()` -> 模型采样。

其中 `clone_history().for_prompt()` 才是实际送给模型的 prompt 历史。

## 2. compact 如何改写历史

当模型切换导致窗口变化，或历史超窗时，compact 会重建压缩历史，并重新注入 initial context。`replace_compacted_history()` 后，`window_generation` 会推进，后续 turn 会基于新的上下文窗口身份继续。

## 3. memory startup 的调用链

memory startup 的主链路是：

`start_memories_startup_task()` -> `phase1::run()` -> `phase2::run()` -> 文件系统同步 / consolidation agent。

它只会在非 ephemeral、启用 MemoryTool、且不是 subagent 的 root session 上启动。

## 4. 关键边界

- turn context 不等于 memory pipeline。
- SQLite state 不等于 `history.jsonl`。
- `memory_summary.md` 是 memory tool 读入口，不等于完整 memory root。
- realtime startup context 与普通 turn context 相关，但不是同一条构造路径。
