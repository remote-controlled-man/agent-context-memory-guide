# `claude code`：存储与状态

## 1. 运行时状态

`bootstrap/state.ts` 里可以看到这套 runtime 关心的核心状态：原始工作目录、project root、session project dir、session 是否禁用持久化等。这些状态决定后续 transcript 和 session-memory 应该落到哪里。

### 源码锚点

- `bootstrap/state.ts:46-46`

## 2. 会话 transcript 的落盘

`utils/sessionStorage.ts` 负责 session transcript 的持久化路径，按 `{projectDir}/{sessionId}.jsonl` 一类布局保存，同时还管理 subagents 目录。它属于历史/会话存储，不直接等于 memory。

### 源码锚点

- `utils/sessionStorage.ts:202-202`

## 3. `session-memory` 路径与权限 carve-out

`utils/permissions/filesystem.ts` 专门为 `session-memory/summary.md` 留了读权限 carve-out。这反过来说明 `SessionMemory` 不是普通 UI 文件，而是 runtime 明确认识的一类内部路径。

### 源码锚点

- `utils/permissions/filesystem.ts:259-261`

## 4. 历史相关存储

- `history.ts` 维护命令历史文件。
- `assistant/sessionHistory.ts` 负责远端 session event 历史分页。

二者都属于 `history` / event 视角，不应与 `memdir` 或 `SessionMemory` 混淆。

### 源码锚点

- `history.ts:114-114`
- `assistant/sessionHistory.ts:1-1`
