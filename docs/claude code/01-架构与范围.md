# `claude code`：架构与范围

## 1. 总体分层

这套实现至少分成四层：

- `context`：决定本轮真正送进模型的系统上下文和用户上下文。
- `memdir / claudemd`：负责跨会话文件型 memory，包括 `MEMORY.md`、daily log、team memory、`CLAUDE.md` 系列文件发现与拼接。
- `SessionMemory`：负责当前 session 的结构化摘要文件 `summary.md`。
- `history / transcript / compact`：负责历史记录、压缩边界、会话落盘和远端事件分页。

## 3. 关键边界判断

- `memdir` 是跨会话、项目级的文件型 memory。
- `SessionMemory` 是当前 session 的摘要文件，不等于长期 memory。
- `history.ts` 记录的是命令历史，不是会话级语义记忆。
- `/context` 展示的是 API 视角上下文，而不是原始 UI 消息数组。
