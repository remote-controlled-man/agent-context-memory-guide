# `letta`：架构与范围

## 1. 四层结构

`letta` 的相关实现可分成四层：

- `AgentState.memory / blocks`：核心记忆。
- `MessageManager + ConversationManager`：消息与会话上下文。
- `ArchiveManager + PassageManager`：长期检索记忆。
- `GitEnabledBlockManager + memory_repo/*`：可选 git-backed memory。

## 3. 关键边界

- core blocks memory 不等于 archive/passage。
- git-backed memory 是 block memory 的可选后端，不是另一种 archive。
- 消息顺序由 `conversation_messages` 关系表维护，不应只看 message 表本身。
