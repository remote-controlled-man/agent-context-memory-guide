# `letta`：存储与状态

## 1. SQL / ORM 层

`letta` 的消息、conversation、archive、passage 等对象都走 ORM / SQL 持久化。与 context 相关的真实顺序由 `conversation_messages` 关系表维护，而不是单看 message 行本身。

### 源码锚点

- `letta/services/conversation_manager.py:613-713`
- `letta/services/message_manager.py:477-1262`

## 2. memfs / git repo / Postgres cache

启用 git-backed memory 后，内存块会写入 memfs 和 git repo，再同步 Postgres cache。`server.py` 会根据配置与 tag 决定是否启用这套路径，并切换对应的 block manager。

### 源码锚点

- `letta/server/server.py:155-161`
- `letta/server/server.py:426-446`
- `letta/services/block_manager_git.py:30-485`

## 3. context window 相关状态

`ContextWindowCalculator` 统计的对象包括 system、core memory、filesystem、tool rules、directories、summary、messages、tool definitions。这说明 `letta` 会把 prompt 结构本身当作状态的一部分来做分段计量。

### 源码锚点

- `letta/services/context_window_calculator/context_window_calculator.py:167-249`

## 4. 记忆仓库路径

本地 memfs 默认落在 `~/.letta/memfs`，并按 `{org_id}/{agent_id}/repo.git` 组织 git repo。skills 目录也会映射成 memory tree 的一部分。

### 源码锚点

- `letta/services/memory_repo/memfs_client_base.py:36-208`
- `letta/services/memory_repo/path_mapping.py:1-31`
